# 🧭 **《Personal RPG System 产品策划案（v1.0）》**

---

# 1. 产品名称（暂定）

- **Personal RPG System**
    
- 简称：**PRS**
    
- 内部代号：**“Persona Engine”**
    

---

# 2. 产品定位

**一个基于本地优先（Local-first）、AI驱动、游戏化的个人成长系统。**  
它通过用户每日输入的日记，自动识别行为、情绪与技能成长，并可视化呈现人生的升级轨迹，像玩一款 RPG 游戏一样记录和强化用户的真实人生。

**这是“人生 OS + AI 私人助理 + 成长可视化工具”的结合。**

---

# 3. 目标用户

### 第一阶段（核心用户）

- 20–40 岁的技术人员、创作者、独立开发者
    
- 自我管理能力强、有长期主义思维
    
- 喜欢游戏化成长
    
- 拥有 AI API Key（愿意构建个人外脑）
    
- 对隐私敏感，不信任中心化 SaaS
    

### 第二阶段（扩展人群）

- 自主学习者
    
- 内容创作者
    
- 独立创业者
    
- 作家、设计师、学生
    
- 使用 Notion、Obsidian 的用户
    

---

# 4. 设计哲学（核心理念）

### 🎯 4.1 本地优先（Local-first）

- 用户数据完全存储在本地 JSON 文件
    
- 不上传任何内容到服务器
    
- 服务端仅负责订阅验证
    
- 所有敏感数据（API Key、日记、技能）均不出本地环境
    

### 🎮 4.2 人生即 RPG（Gamification）

- 技能点
    
- 属性面板
    
- 六边形图
    
- 成长曲线
    
- 成就系统
    
- 每日升级
    
- 多角色存档
    

### 🧠 4.3 AI 作为人生伴生引擎（Life Companion Engine）

- 自动解析日记
    
- 归类行为 → 技能经验
    
- 自动检测学习行为、项目完成、突破点
    
- 自动生成成长记录
    
- 自动生成复盘报告
    

### 🗃️ 4.4 用户完全拥有自己的数据（Data Ownership）

- JSON 可导入导出
    
- GitHub 私库同步
    
- 多存档管理
    
- 可手动编辑、可被外部程序扩展
    

### ⚙️ 4.5 扩展性

- 技能点可自定义
    
- 行为模式可扩展
    
- 经验算法可自定义
    
- 主题与 UI 可替换
    
- 未来可连接更强大的模型
    

---

# 5. 系统核心价值

- 可视化成长，让人生“可玩、可见、可管理”
    
- AI 解析真实行为 → 转化为成长点
    
- 长期数据沉淀，成为用户人生档案
    
- 让用户每天见到自己的成长而不焦虑
    
- 成为用户的 **第二大脑 + 外部人生记录者**
    
- 极强隐私保护（差异化竞争点）
    

---

# 6. 系统核心功能（概览）

### 🧩 6.1 初始化（角色创建）

- 表单填写个人信息：年龄、职业、目标、兴趣等
    
- AI 生成初始属性面板
    
- 用户选择角色模板（Creator / Engineer / Entrepreneur / Hybrid 等）
    
- 用户定义技能点（一级、二级技能）
    
- 生成初始六边形图
    

---

### 📒 6.2 每日日记（核心循环）

用户每天输入一段自然语言，如：

> 今天研究了Unity行为树，修复了PhotoLens推广页的bug，还读了两章有关认知心理学的内容。

系统自动：

1. NLP 分析行为
    
2. 分类到技能点
    
3. 计算经验值
    
4. 更新技能等级
    
5. 更新主属性
    
6. 更新可视化面板
    
7. 保存 JSON
    

---

### 📊 6.3 可视化大屏

- 六边形图
    
- 技能点列表（经验条）
    
- 成长曲线图
    
- 今日成长动态
    
- 成就解锁提示
    
- 主角等级变化
    

---

### 🧠 6.4 AI 复盘与成长建议

周期性输出：

- 每周总结
    
- 每月总结
    
- 趋势变化
    
- 成长盲点
    
- 技能偏科提醒
    
- 建议下一步重点
    

---

### 📦 6.5 多存档系统

- 多角色（不同人生方向，可以并行体验）
    
- 存档切换
    
- 存档备份/恢复
    
- 适合创业者、学生、创作者多线成长
    

---

### 💾 6.6 JSON 导出 / 导入

- 一键导出全部数据（zip）
    
- 单项导出（技能树、配置、日志）
    
- 导入旧数据（合并或覆盖）
    

---

### ☁️ 6.7 GitHub 私库同步（可选）

- 自动 push JSON
    
- 保留版本历史（commit）
    
- 全端同步
    
- 不涉及你的服务器
    

---

### 🔐 6.8 API Key 本地管理

- 用户自己输入
    
- LocalStorage 或 IndexedDB 加密保存
    
- 从不发往服务器
    

---

### ⚙️ 6.9 自定义经验公式

- 行为强度
    
- 行为时长
    
- 行为类型（学习/实践/推广/写作）
    
- 难度系数
    
- 角色模板加权
    

---

# 7. 核心循环（Daily Loop）

```
用户输入日记
→ AI 分析行为、情绪、成果
→ 系统映射到技能点
→ 计算经验值
→ 升级技能点
→ 更新主属性
→ 更新六边形图
→ 保存本地 JSON
→ 显示今日成长
→ 第二天继续
```

这是产品每天驱动用户继续使用的关键循环。

```
  }

```

# 11. 技术架构（Local-first）

### ✔ 前端

- React / Next.js
    
- 本地 IndexedDB 或 LocalStorage
    
- 状态管理 Recoil 或 Zustand
    
- ECharts / D3.js 做可视化
    

### ✔ 本地数据

- IndexedDB
    
- JSON 文件导入导出
    

### ✔ AI 模型调用

- 用户 API Key
    
- 前端直接调用 OpenAI GPT-5.1 API
    
- 不经过你的服务器
    

### ✔ 服务端

- 只处理：Stripe/Paddle 订阅
    
- 不保存用户名、技能、日志等隐私
    

架构非常干净、安全、无风险。

# 17. 风险与规避方案

### 风险 1：用户担心复杂度

→ 用角色模板降低学习成本

### 风险 2：用户怕丢数据

→ 自动备份 + GitHub 同步

### 风险 3：隐私担心

→ 明确说明所有内容仅存储本地

### 风险 4：早期功能太多

→ 从 MVP 开始

### 风险 5：AI 解析不精准

→ 用户可手动修改技能点、经验
