# 路由策略

他是一种可以帮助过滤路由，修改属性的工具
- 使用节点结构，每一种节点都有 if-match, apply, 有点像编程中的if-else 和 方法函数
- 可以对路由的属性做修改
- 有 permit 和 deny两种方式

```
# 定义route-policy
route-policy FILTER_ROUTE permit node 10
 if-match ip-prefix PREFIX_LIST
 if-match cost 10
 apply preference 150
 apply tag 100

route-policy FILTER_ROUTE deny node 20
 if-match ip address prefix-list DENY_LIST
```

使用场景：BGP路由控制

# 过滤策略

基于ACL规则或IP前缀的路由过滤
- import：过滤接收的路由
- export：过滤发布的路由

# 路由重分发

就是把学习到路由信息注入到另一个路由协议，这是为了解决不同路由协议之间无法相互通信的问题

例如有路由器R1, R2, R3, R1运行OSPF，R2运行BGP，为了把BGP中的路由信息转发到R1，这个时候有一台R3， 它运行OSPF和BGP，这个时候就把它从R2中学习到的路由信息注入到OSPF中，再转发给R1，此时R3起到一个ASBR路由器的作用
